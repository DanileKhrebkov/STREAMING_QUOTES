# Streaming Quotes

Клиент-серверная система для стриминга биржевых котировок с поддержкой множественных подписок и keep-alive механизмом.

## Сборка проекта

```bash
cargo build
```

Для сборки в оптимизированном режиме:

```bash
cargo build --release
```

## Запуск

### Сервер

Сервер запускается на TCP порту 7878 и принимает команды подписки от клиентов.

```bash
cargo run --bin server
```

Пример вывода:
```
🚀 Starting Quote Server v1.0
📡 TCP port: 7878
📊 Loaded 10 tickers from file
💓 PING listener on port 45678
🔄 Quote generator started
📤 Quote distributor started
💓 PING handler started
⏰ Timeout checker started
✅ Server is ready and listening
```

### Клиент

Клиент подключается к серверу и запрашивает стриминг котировок для указанных тикеров.

```bash
cargo run --bin client -- <server_addr> <local_udp_port> <tickers_file>
```

**Аргументы:**
- `server_addr` - TCP адрес сервера (например, `127.0.0.1:7878`)
- `local_udp_port` - локальный UDP порт для приема котировок
- `tickers_file` - путь к файлу со списком тикеров

**Пример:**
```bash
cargo run --bin client -- 127.0.0.1:7878 9000 assets/tickers.txt
```

Пример вывода клиента:

## Формат файла тикеров

Файл с тикерами должен содержать один тикер на строку:

- Пустые строки игнорируются
- Пробелы по краям автоматически обрезаются
- Регистр символов важен (тикеры чувствительны к регистру)

**Пример файла `assets/tickers.txt`:**
```
AAPL
MSFT
TSLA
GOOGL
AMZN
META
NVDA
JPM
V
WMT
```

## Проверка работы

### 1. Проверка фильтрации тикеров

**Сценарий:** Клиент запрашивает только определенные тикеры, сервер отправляет только их.

**Шаги:**
1. Создайте файл `assets/test_tickers.txt` с содержимым:
```
AAPL
TSLA
```

2. Запустите сервер:
```bash
cargo run --bin server
```

3. Запустите клиент с фильтром:
```bash
cargo run --bin client -- 127.0.0.1:7878 9000 assets/test_tickers.txt
```

**Ожидаемый результат:** В консоли клиента появляются только котировки AAPL и TSLA. Котировки других тикеров (MSFT, GOOGL и т.д.) не отображаются.

### 2. Проверка тайм-аута PING

**Сценарий:** Сервер автоматически отключает подписку, если от клиента не приходит PING в течение 5 секунд.

**Шаги:**
1. Запустите сервер:
```bash
cargo run --bin server
```

2. Запустите клиент:
```bash
cargo run --bin client -- 127.0.0.1:7878 9000 assets/tickers.txt
```

3. Дождитесь появления котировок, затем остановите клиент (Ctrl+C)

**Ожидаемый результат:** В логах сервера через 5 секунд появляется сообщение:
```
⏰ Removing subscription for 127.0.0.1:9000 (timeout)
```

Котировки перестают отправляться на этот UDP адрес.

### 3. Проверка множественных подписок

**Сценарий:** Два клиента с разными наборами тикеров получают разные котировки.

**Шаги:**
1. Запустите сервер:
```bash
cargo run --bin server
```

2. В терминале B запустите клиент 1:
```bash
cargo run --bin client -- 127.0.0.1:7878 9000 assets/tickers.txt
```

3. В терминале C запустите клиент 2 (другой порт и файл):
```bash
cargo run --bin client -- 127.0.0.1:7878 9001 assets/test_tickers.txt
```

**Ожидаемый результат:** Оба клиента получают котировки независимо друг от друга. Клиент 1 получает все тикеры, клиент 2 получает только AAPL и TSLA.

### 4. Проверка обработки ошибок

| Действие | Ожидаемый ответ |
|----------|-----------------|
| Отправить мусор вместо команды | `ERR invalid command` |
| Указать битый UDP адрес | `ERR Invalid UDP address` |
| Запросить пустой список тикеров | `ERR Empty ticker list` |
| Запросить неизвестный тикер | `ERR unknown ticker: UNKNOWN` |

**Пример проверки через netcat:**
```bash
# Подключение к серверу
nc 127.0.0.1 7878

# Отправка некорректной команды
INVALID COMMAND
# Ответ: ERR invalid command

# Отправка команды с неизвестным тикером
STREAM 127.0.0.1:9000 UNKNOWN
# Ответ: ERR unknown ticker: UNKNOWN

# Отправка корректной команды
STREAM 127.0.0.1:9000 AAPL,TSLA
# Ответ: OK
```

## Протокол обмена

### TCP (Команды управления)

**Команда подписки:**
```
STREAM <udp_addr> <ticker1>,<ticker2>,...
```

**Ответы сервера:**
- `OK` - успешная подписка
- `ERR <message>` - ошибка с пояснением

### UDP (Данные)

**Формат котировки:**
```
<ticker>|<price>|<volume>|<timestamp_ms>
```

**Пример:**
```
AAPL|150.25|2345|1234567890
```

**Keep-alive сообщение:**
```
PING
```

Клиент отправляет PING на UDP адрес сервера каждые 2 секунды.

## Известные ограничения

1. **Только IPv4**: Сервер и клиент работают только с IPv4 адресами. Поддержка IPv6 не реализована.

2. **Локальный хост**: Для корректной работы все компоненты должны запускаться на localhost (127.0.0.1) или в пределах одной сети.

3. **Одна команда на TCP соединение**: TCP соединение принимает только одну команду STREAM. После получения команды и отправки ответа соединение ожидает закрытия. Повторная отправка команд в том же соединении не поддерживается.

4. **Фиксированный порт сервера**: Сервер всегда запускается на порту 7878. Изменение порта требует изменения константы в коде.

5. **Ограничение на количество подписок**: Максимальное количество одновременных подписок ограничено только системными ресурсами и настройками файловых дескрипторов.

6. **Генерация котировок**: Котировки генерируются один раз в секунду для одного случайного тикера. Обновление цен использует модель случайного блуждания.

7. **Без сохранения состояния**: При перезапуске сервера все подписки теряются. История котировок не сохраняется.

8. **Обработка Ctrl+C**: Принудительное завершение клиента или сервера через Ctrl+C может привести к неполной очистке ресурсов, но не влияет на стабильность системы при последующих запусках.

## Технические детали

### Конфигурационные параметры

| Параметр | Значение | Описание |
|----------|----------|----------|
| TCP порт | 7878 | Порт для команд управления |
| UDP буфер | 2 KiB | Размер буфера для UDP сообщений |
| PING интервал | 2 сек | Интервал отправки PING клиентом |
| PING таймаут | 5 сек | Время без PING до отключения подписки |
| Генерация котировок | 1 сек | Частота генерации новых котировок |

### Модель потоков

**Сервер:**
- Главный поток: прием TCP соединений
- Поток генератора: генерация котировок
- Поток рассыльщика: распределение котировок подписчикам
- Поток PING: прием и обработка PING сообщений
- Поток таймаутов: проверка и удаление неактивных подписок
- Отдельный поток на каждое TCP соединение

**Клиент:**
- Главный поток: прием и вывод UDP котировок
- Отдельный поток: отправка PING сообщений

## Устранение неполадок

### Сервер не запускается
- Проверьте, что порт 7878 не занят другим приложением
- Убедитесь, что файл `assets/tickers.txt` существует и содержит тикеры

### Клиент не подключается
- Проверьте, что сервер запущен
- Убедитесь, что указан правильный адрес сервера
- Проверьте файрволл (может блокировать соединение)

### Котировки не приходят
- Проверьте, что UDP порт не занят другим приложением
- Убедитесь, что в команде STREAM указан правильный порт
- Проверьте, что клиент запрашивает существующие тикеры

### PING не работает
- Проверьте, что клиент и сервер используют одинаковый UDP адрес
- Убедитесь, что UDP порт сервера для PING доступен
